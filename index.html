Sorting Algorithms Explained

Bubble Sort

Like bubbles rising in water, larger elements "bubble up" to the end by repeatedly comparing and swapping adjacent elements.

Time Complexity: O(n²)
Space Complexity: O(1)

Selection Sort

Like finding the smallest card in a hand - repeatedly finds the minimum element and puts it at the beginning.

Time Complexity: O(n²)
Space Complexity: O(1)

Insertion Sort

Like sorting playing cards - builds the final array one item at a time by inserting each element in its correct position.

Time Complexity: O(n²)
Space Complexity: O(1)

Quick Sort

Divide and conquer - picks a "pivot" element and partitions the array around it, then recursively sorts the partitions.

Time Complexity: O(n log n)
Space Complexity: O(log n)

Merge Sort

Divide and conquer - divides the array into halves, sorts them, and then merges the sorted halves back together.

Time Complexity: O(n log n)
Space Complexity: O(n)
